# Документация по доступности (Accessibility)

## Общие принципы

Проект соответствует стандарту WCAG 2.1 Level AA. Доступность важна для обеспечения равного доступа к контенту для всех пользователей, включая людей с ограниченными возможностями.

## Клавиатурная навигация

### Глобальные горячие клавиши

- `Tab` - Навигация вперед по интерактивным элементам
- `Shift + Tab` - Навигация назад по интерактивным элементам
- `Enter` - Активация кнопок и ссылок
- `Escape` - Закрытие модальных окон и меню
- `Ctrl/Cmd + K` - Переход к основному контенту (skip navigation)

### Навигация по секциям

Используйте меню навигации для перехода между секциями:

- About
- Experience
- Portfolio
- Stack
- Contact

## Поддержка скринридеров

Сайт протестирован со следующими инструментами:

- NVDA (Windows)
- JAWS (Windows)
- VoiceOver (macOS/iOS)
- TalkBack (Android)

### Лендмарки

Страница использует семантические HTML5 лендмарки:

- `<header>` - Заголовок сайта и навигация
- `<main>` - Основной контент
- `<nav>` - Меню навигации
- `<section>` - Секции контента
- `<form>` - Контактная форма

### ARIA атрибуты

Все интерактивные элементы имеют описательные ARIA атрибуты для скринридеров.

## Визуальная доступность

### Контраст цветов

Весь текст соответствует стандартам WCAG AA:

- Нормальный текст: соотношение контрастности 4.5:1
- Крупный текст: соотношение контрастности 3:1

### Индикаторы фокуса

Все интерактивные элементы имеют видимые индикаторы фокуса при навигации с клавиатуры.

### Темная/Светлая тема

Обе темы сохраняют стандарты доступности.

## Снижение анимации

Сайт учитывает системную настройку `prefers-reduced-motion`:

- Анимации уменьшены или удалены
- Переходы мгновенны

## Структура HTML

### Семантическая разметка

- Используются подходящие заголовки (h1-h6) в правильной иерархии
- Списки обозначены тегами `<ul>`, `<ol>`, `<li>`
- Структурные элементы имеют соответствующие роли

### Обработка форм

- Все поля формы имеют связанные `<label>` или `aria-label`
- Ошибки форм обозначены с помощью `aria-invalid` и `aria-describedby`
- Сообщения об ошибках и успехе используют `role="alert"` и `aria-live="assertive"`

## Компоненты с доступностью

### Header

- Ссылка "Skip to main content" для быстрой навигации
- ARIA-атрибуты для мобильного меню
- Правильное описание навигационных элементов

### Modal

- Фокус остается внутри модального окна
- Закрытие по клавише Escape
- Блокировка скролла основной страницы
- Правильные ARIA-роли (`role="dialog"`, `aria-modal="true"`)

### Contact Form

- Правильные ARIA-атрибуты для полей ввода
- Связывание ошибок с полями формы
- Индикаторы загрузки с `aria-busy`
- Сообщения с `role="status"` для вспомогательных технологий

## Технические реализации

### useFocusTrap хук

Управление фокусом внутри модальных окон, чтобы фокус не уходил за пределы компонента при навигации с клавиатуры.

### useSkipNavigation хук

Функциональность "skip navigation" по нажатию `Ctrl/Cmd + K`.

### sr-only утилиты

CSS классы для скрытия элементов визуально, но оставления их доступными для скринридеров.

## Тестирование доступности

### Запуск тестов

```bash
npm run test:a11y
```

### Используемые инструменты

- axe DevTools
- WAVE
- Lighthouse
- jest-axe (для автоматических тестов)

### Автоматические тесты

- Проверка нарушений WCAG в компонентах
- Тесты для Header, ContactSection, Button
- Проверка соответствия стандартам

## Аудит доступности

### Регулярные проверки

- Проверка с помощью axe-core
- Тестирование с разными скринридерами
- Проверка на реальных устройствах
- Ручная проверка клавиатурной навигации

### Метрики

- Lighthouse accessibility > 95
- Отсутствие серьезных нарушений axe-core
- Полная клавиатурная навигация
- Поддержка скринридеров

## Рекомендации по разработке

- Всегда добавляйте `alt` атрибуты к изображениям
- Используйте семантические HTML теги
- Проверяйте цветовой контраст
- Добавляйте ARIA атрибуты при необходимости
- Пишите тесты доступности для новых компонентов
- Проверяйте компоненты с клавиатуры
- Учитывайте пользователей с ограниченными возможностями при проектировании
